1 KAFKA Playground

1.1 Starting a kafka cluster

  • Clone wurstmeister/kafka-docker
$ git clone https://github.com/wurstmeister/kafka-docker.git
  • Update docker-compose.yml with your docker host IP (KAFKAADVERTISEDHOSTNAME)
version: '2'
services:
  zookeeper:
    image: wurstmeister/zookeeper
    ports:
      - "2181:2181"
  kafka:
    build: .
    ports:
      - "9092"
    environment:
      KAFKA_ADVERTISED_HOST_NAME: 172.19.0.1
      KAFKA_ZOOKEEPER_CONNECT: kafka-docker_zookeeper_1:2181
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
  • Start the cluster
$ docker-compose up
  • Scale the cluster
$ docker-compose scale kafka=2
  • List Containers
$ docker-compose ps
Name Command State Ports
kafka-dockerkafka1 start-kafka.sh Up 0.0.0.0:32928->9092/tcp
kafka-dockerkafka2 start-kafka.sh Up 0.0.0.0:32929->9092/tcp
kafka-dockerzookeeper1 /bin/sh -c /usr/sbin/sshd Up 0.0.0.0:2181->2181/tcp, 22/tcp, 2888/tcp, 3888/tcp

1.2 Playing with console consumers and producers

Figure 1: Network Overview

  • Kafka Shell
# replace with your IP
$ start-kafka-shell.sh <DOCKER_HOST_IP> <ZK_HOST:2181>
  • Create a topic test with 4 partitions and replica factor 2
$ $KAFKA_HOME/bin/kafka-topics.sh --create --topic topic --partitions 4 --zookeeper $ZK --replication-factor 2
$ $KAFKA_HOME/bin/kafka-topics.sh --describe --topic topic --zookeeper $ZK
Topic: topic PartitionCount:4 ReplicationFactor:2 Configs:
Topic: topic Partition: 0 Leader: 1001 Replicas: 1001,1002 Isr: 1001,1002
Topic: topic Partition: 1 Leader: 1002 Replicas: 1002,1001 Isr: 1002,1001
Topic: topic Partition: 2 Leader: 1001 Replicas: 1001,1002 Isr: 1001,1002
Topic: topic Partition: 3 Leader: 1002 Replicas: 1002,1001 Isr: 1002,1001
  • Create a console producer
$ $KAFKA_HOME/bin/kafka-console-producer.sh --topic=topic --broker-list=`broker-list.sh`
  • Create a console consumer
$ $KAFKA_HOME/bin/kafka-console-consumer.sh --bootstrap-server=`broker-list.sh` --topic=topic
  • Check the consumer groups
$ $KAFKA_HOME/bin/kafka-consumer-groups.sh --bootstrap-server=`broker-list.sh` --list
  • Describe the consumer group
$ $KAFKA_HOME/bin/kafka-consumer-groups.sh --bootstrap-server=`broker-list.sh` --group=console-consumer-17193 --describe
